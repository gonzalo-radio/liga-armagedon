# CONTRIBUTING

## Cómo contribuir

Este proyecto se desarrolla mediante iteraciones. Cada cambio debe ser pequeño, revisable y versionado.

## Flujo de trabajo

```
Idea
  ↓
RFC (propuesta en rfcs/)
  ↓
Discusión
  ↓
Aprobación
  ↓
Incorporación al reglamento
  ↓
Nueva versión
```

## Directrices

1. **Markdown**: todo el contenido se escribe en Markdown.
2. **Repositorio como fuente única de verdad**: no se trabaja sobre documentos de Word ni versiones dispersas.
3. **Cambios pequeños**: cada cambio debe ser atómico y revisable.
4. **Tres tipos de documento**:
   - **Reglamento** (`docs/`): solo las reglas que leerán los jugadores.
   - **Notas del Warmaster** (en `docs/`): cómo organizar la campaña y resolver situaciones especiales.
   - **Notas de diseño** (`design/`): por qué existe cada regla, para facilitar revisiones futuras.
